[0:20 — WHAT IT IS]
DevForge AI is built using the Google Agent Development Kit 2.0. Instead of just one AI model giving you a basic answer, DevForge AI uses a structured graph workflow that coordinates a team of specialized AI agents. Together, they analyze project ideas, estimate resources, write API specifications, check security parameters, and require human-in-the-loop approval before final output.

[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's look at the workflow architecture. Everything starts when the user provides their product description.
(pause)
First, the input runs through a Security Checkpoint node. This scans for malicious prompt injections, scrubs sensitive PII, and runs a domain-specific check to ensure the request is tech-related. If it fails, it routes to a terminal Security Event block.
(pause)
If it passes, it moves to the CEO Orchestrator agent. The CEO acts as the brain, delegating tasks to our two specialized sub-agents: the Product Manager and the Software Architect.
(pause)
The Product Manager agent analyzes the scope and calls an external Model Context Protocol tool to estimate the total development hours.
(pause)
The Software Architect agent designs database schemas and API endpoints, calling another MCP tool to recommend a folder structure and validate layout cleanliness.
(pause)
Once they both complete, the CEO merges their results and forwards the software package to the Human Review node. Here, the graph pauses, requiring an engineer's sign-off before proceeding. Once approved, the Final Output node renders the structured package.
